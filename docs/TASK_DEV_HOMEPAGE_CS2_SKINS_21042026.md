**Приоритет:** HIGH (большой ассортимент готовых товаров, никак не представлен на главной)
**Дата:** 21.04.2026
**Автор:** Кир (CEO)

## Контекст и проблема

На gaming-goods.ru есть **~80 активных брендов** связанных со скинами CS2, кейс-сайтами и пополнением криптокошельков для skin-trading. Средний ассортимент: 5-20 товаров на бренд, **всего 400+ активных products**. Выручка по этой категории пока незначительная, но **это быстрорастущий рынок** (молодая аудитория 14-25, high-frequency transactions).

**Текущий статус на главной (`/`):** ссылок на эту категорию **нет вообще**. 11 существующих H2-секций покрывают все главные вертикали (игры, мобильные, соцсети, ИИ), но CS2 skins — пропущен.

**Бизнес-контекст:** Kinguin (главный саплаер) имеет [отдельный портал `cs2.kinguin.net`](https://cs2.kinguin.net/) — значит рынок зрелый и прибыльный. Наша задача — дать пользователям простой путь навигации к этим товарам.

## Технические требования

### Источник данных

**НЕ хардкодить** список URL. Использовать существующий механизм секций главной (тот же что для «Хиты продаж», «Игры», «Популярные игры» и т.п.). Если сейчас на главной список брендов берётся:
- **Из БД-таблицы секций** (типа `home_sections` с `section_name`, `brand_slugs`) — создать новую запись с этими 23 slug'ами
- **Из хардкода в Nuxt SSR-компоненте** — добавить массив туда же, следуя паттерну существующих секций

Бэкенд уже знает как по slug'у `csgo500-gift-card` достать:
- title (будет «CSGO500 $3 — купить подарочную карту | Гейминг Гудс»)
- иконку/изображение бренда (если есть в CDN)
- категорию (Gift Card)
- ссылку на `/t/{slug}`

Всё уже работает для других секций, значит достаточно добавить **один конфиг-источник с 23 slug'ами**, разбитыми на 3 подгруппы.

### UI/UX

Следовать стилю существующих секций. Не изобретать новый layout:
- Сетка из карточек 4-6 в ряд на десктопе, горизонтальный скролл на мобильном
- Карточка: логотип/иконка + название бренда + короткий подзаголовок (типа «открытие кейсов», «trading», «пополнение») + цена «от ~X₽» если API возвращает
- H2 заголовок: `Скины CS2 и пополнение кейс-сайтов`
- Подзаголовок под H2 (опционально): «Подарочные карты для топ-50 брендов CS2-скинов: открывайте кейсы, торгуйте, пополняйте балансы. Моментальная доставка кода на e-mail.»
- Ссылка «Все бренды CS2 →» ведёт на новую агрегаторную страницу или категорию (если её нет — создать как расширение в будущем)

### Приоритет вывода на мобильных

На мобильном экране вертикальный скролл и компактные 2 в ряд карточки. Не использовать горизонтальный карусель если это нагружает LCP — показывать первые 6 сразу, кнопку «Показать ещё».

### SEO metadata для главной

Если секция влияет на meta-description главной — добавить упоминание:
- `description` главной: текущее + «…подарочные карты для Skin.Club, Hellcase, DatDrop, CSGO500 и других CS2-кейс-сайтов.»
- Ключевые слова: `скины cs2 купить, кейс сайт cs2, подарочные карты hellcase, cs2 pin status upgrade, пополнить csgo500`

### Аналитика

Добавить в GA4/Метрику тегирование кликов по новой секции:
- Event name: `home_section_click`
- Params: `section_name="cs2_skins"`, `subsection="cases" | "trading" | "games"`, `brand_slug="..."`

Это поможет через 2 недели измерить конверсии и понять какие из 23 брендов работают лучше всего.

## Acceptance criteria

1. На главной `/` после секции «Мобильные игры» видна секция «Скины CS2 и пополнение кейс-сайтов» с 3 подгруппами (кейсы / trading / игры)
2. В каждой карточке — clickable-ссылка на соответствующую `/t/{slug}` страницу
3. Все 23 slug'а открываются без 404, содержимое SSR рендерится
4. Мобильная версия не ломает main LCP (< 2.5s)
5. События `home_section_click` фиксируются в GA4/Метрике
6. В `sitemap.xml` ничего не меняется (страницы `/t/...` уже там)
