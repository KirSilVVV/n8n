# SEO: Диагностика каннибализации /t/ vs /marketplace/
**gaming-goods.ru | 05.03.2026**

---

## Вердикт: Google продвигает /marketplace/ — это ПРОБЛЕМА

| Тип URL | Страниц в GSC | Кликов | Показов | Поз avg | CTR avg |
|---------|--------------|--------|---------|---------|---------|
| `/t/` (биржа) | **2** | 47 | 1 148 | 7.4 | 6.9% |
| `/marketplace/` | **16** | 483 | 8 560 | 8.2 | 8.9% |
| Главная + остальные | 2 | 415 | 5 330 | 8.4 | 14.5% |

**Google почти полностью игнорирует /t/ — в индексе только 2 страницы из всего каталога биржи.**

---

## Nutaku — конкретный конфликт

| URL | Клики | Показов | Позиция | CTR |
|-----|-------|---------|---------|-----|
| `/t/nutaku-currency` (нужный!) | **32** | **1 006** | **6.7** | 3.2% |
| `/marketplace/nutaku` (ненужный) | 16 | 1 228 | 7.2 | 1.3% |

**Ситуация:** оба URL показываются по одним запросам. Google сам выбирает какой продвигать.
Пока `/t/nutaku-currency` в топе (поз 6.7 vs 7.2) — но `/marketplace/nutaku` набирает **больше показов**.
Это значит Google экспериментирует и может переключиться.

**Риск:** если Google решит что `/marketplace/nutaku` — "правильная" страница,
`/t/nutaku-currency` выпадет из показов. При 1 006 показов/мес это потеря ~32 кликов/мес = ~3-4 продажи.

## Полная картина /t/ страниц

Из всего каталога биржи в GSC видны только **2 страницы**:

| URL | Клики | Показов | Позиция | CTR |
|-----|-------|---------|---------|-----|
| `/t/nutaku-currency` | 32 | 1 006 | 6.7 | 3.2% |
| `/t/fnaf-world-game-key` | 15 | 142 | 8.0 | 10.6% |

**Вывод: 18 000+ биржевых карточек `/t/` не проиндексированы.** Google видит только те 2, которые
случайно получили внешние ссылки или попали в sitemap.

## Что делать — приоритет по срочности

### 🔴 СРОЧНО: Canonical для Nutaku

На `/marketplace/nutaku` добавить canonical тег указывающий на `/t/nutaku-currency`:
```html
<link rel="canonical" href="https://gaming-goods.ru/t/nutaku-currency" />
```

Это скажет Google: "продвигай /t/, /marketplace/ — техническая страница".

**Задача Саше:** добавить canonical в шаблон маркетплейсовых страниц, когда у категории
есть соответствующая биржевая карточка `/t/`.

---

### 🔴 СРОЧНО: Sitemap только с /t/ URL

Текущая проблема: Google не знает о существовании 18K+ биржевых карточек.
Решение — sitemap.xml содержащий все `/t/` URL с `<priority>0.8</priority>`.

**Источник данных:** PostgreSQL — таблица продуктов, поле slug → URL вида `/t/{slug}`

SQL запрос для sitemap (Саше не нужен — можно сделать через n8n + PostgreSQL Tool):
```sql
SELECT slug, updated_at 
FROM products 
WHERE active = true 
ORDER BY updated_at DESC
```

---

### 🟡 ВАЖНО: /marketplace/ страницы с noindex или canonical

Для всех `/marketplace/` страниц которые дублируют `/t/` — добавить canonical на `/t/`.
Для `/marketplace/` страниц без биржевого аналога — оставить как есть.

## Indexing Bot (WF-204) — что нужно проверить

Перед построением бота нужно понять структуру URL в БД:

**Проверяю через PostgreSQL:**
- Есть ли таблица `products` с полем `slug`?
- Какой формат: `/t/{slug}` или `{slug}` без префикса?
- Сколько активных продуктов?

**Google Indexing API:**
- Квота: 200 URL/день (бесплатно для верифицированных владельцев)
- Нужен отдельный Service Account с правами на Indexing API
- Приоритет: новые/обновлённые продукты + страницы с 0 показов в GSC

**Яндекс Webmaster:**
- Endpoint для submission: `POST /v4/user/{userId}/hosts/{hostId}/recrawl/queue`
- Квота: 10 000 URL/день
- Токен уже есть (OAuth)
